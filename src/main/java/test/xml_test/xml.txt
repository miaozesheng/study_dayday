关于xml相关
    XML 在有些业务场景下线,存储数据或者传输数据给别人的时候,数据给别人の时候,数据需要给别人的时候,数据需要满足一定的规范.
    XML解析技术 xml文件中存储的数据需要提出出来,使用该技术
    Xpath 如何方便的在xml文件中进行数据的检索

    XML
        xml是可扩展标记语言 extensible markup language 表示的是一种数据格式 可以描述非常复杂的数据结构 常用于存储和传输数据

        特点:
            1.纯文本,使用utf-8编码,可以嵌套多个标签
            2.如果把xml内容存为文件,那么它就是一个XML文件
            3.xml的使用场景:京城被当成消息进行网络传输,或者作为配置文件用于存储系统的消息 后缀名为.xml

        什么是文档约束?
            用来限定xml文件中的标签及属性应该怎么写,一次强制约束程序员必须按照文档约束的规定来编写xml文件
        文档约束的分类
            1.DTO
                自己写一个dtd约束文档,后缀必须是.dtd结尾
                在需要编写的xml文件中,导入该dtd约束文档
                然后按照约束规定编写xml文件内容

                dto约束可以约束xml文件的编写,但是不能约束具体的数据类型

            2.schema
                schema可以约束具体的数据类型,约束能力上更强大
                schema本身就是一个xml文件,本省也受到其他约束文件的约束,所以编写的更加严身

                我们可以使用schema文档约束,约束一个xml文件的编写 schema文档的后缀必须是.xsd XML Schema Definition

    XML解析技术
        1.SAX解析 单行解析
        2.DOM解析 文件全部读进内存,整体解析
            JAXP sun公司提供的xml解析api
            JDOM JDOM是一个开源项目,基于树型结构,利用纯Java技术对xml文旦实现解析 生成 序列化以及多种操作
            dom4j 是JDOM的升级版,用来读写xml文件,性能友谊 功能强大 易于使用 也是开源代码
            jsoup 功能强大DOM方式的xml解析开发包,尤其对HTML解析更加方便

       使用dom4j解析出xml文件
       1.下载dom4j框架 引入jar包

       举例说明:
            Document对象 整个xml文档
            Element对象 标签
            Attribute对象 属性
            Text 文本

    XML检索技术
        只是用来检索信息,不会用来解析信息
        xpath使用路径表达式来定位xml文档中的元素节点或者属性节点
        xpath依赖于dom4j,另外还需要导入xpath的jar jaxen-1.1.1

        Xpath的四大检索方法:
            1.绝对路径
                /根元素/子元素/孙元素 从根元素开始,一级一级向下查找,不能跨级
            2.相对路径
            3.全文检索
            4.属性查找
